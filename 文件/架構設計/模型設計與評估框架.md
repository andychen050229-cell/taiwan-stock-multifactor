# 模型設計與評估框架

## 雙軌模型設計

### A 軌：Predictive Engine（預測引擎）

#### 模型優先順序（由簡至繁）
1. **Logistic Regression**：方向分類 baseline
2. **Random Forest**：非線性 baseline
3. **XGBoost / LightGBM**：主力模型（推薦第一版核心）
4. **Ranking Model**：LambdaMART 排序模型
5. **FinBERT + Classifier**：文本情緒分類
6. **LSTM / Transformer**：序列模型（後期考慮）

#### 每個 Horizon 各自訓練
- 短線 5D：二元分類（漲/跌）+ 機率校準
- 中線 20D：排序（優於/劣於大盤）+ 連續分數
- 長線 60D：多分類（強烈看好/看好/中性/看淡/強烈看淡）

### B 軌：Interpretable Decision Layer（決策轉譯層）

#### 設計形式
- Grouped SHAP → 五大支柱貢獻度
- Monotonic GAM → 可解釋評分卡
- Rule-based mapping → 標準化決策語言

#### 輸出標籤
| 標籤 | 意義 |
|------|------|
| 可進場 | 多數訊號偏多、成本可控、風險可接受 |
| 可觀察 | 有潛力但部分條件未滿足 |
| 不宜追價 | 方向或許偏多但短線過熱或成本過高 |
| 減碼觀望 | 訊號轉弱、風險升高 |
| 暫時避開 | 風險過高或反證過多 |

## 訊號淘汰規則（Kill Criteria）

1. 成本後無邊際增益 → 淘汰
2. 只在單一子期間有效 → 降權或淘汰
3. 與既有訊號高度重複 → 合併或淘汰
4. 解釋不穩定（重要性翻轉） → 淘汰
5. 覆蓋率不足 → 不作為主訊號
6. 對 universe/參數極敏感 → 視為 overfit
7. 報告層無法追溯 → 禁止上線
