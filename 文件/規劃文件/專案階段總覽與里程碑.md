# 股市趨勢預測導向決策輔助系統：專案階段總覽與里程碑

## 專案總時程：14–18 週（4–5 人團隊）

---

## 第一階段：需求確認與系統架構設計（第 1–2 週）
- 確認專案範圍、目標使用者、市場範圍
- 撰寫系統需求規格書
- 設計系統架構圖（前端/後端/資料流）
- 決定技術棧與部署方案
- **KPI**：需求文件完成度 100%、架構圖通過團隊審閱

## 第二階段：資料來源調查與取得（第 3–4 週）
- 建立台股/美股資料取得 Pipeline
- API 串接（Yahoo Finance / FinMind / Alpha Vantage）
- 資料品質初步檢驗
- **KPI**：資料完整率 > 95%、延遲 < 24 小時、覆蓋至少 200 支標的

## 第三階段：資料清洗與特徵工程（第 5–6 週）
- Point-in-time 對齊
- 五大主支柱的特徵提取
- 共線性檢測與訊號打包
- 多 horizon 標籤設計（5D/20D/60D）
- **KPI**：特徵數量 50–80 個、共線性 < 0.7、覆蓋率 > 98%

## 第四階段：Baseline 模型建立（第 7–8 週）
- Logistic Regression / Random Forest baseline
- 三個時間維度各一組模型
- 初步樣本外測試
- **KPI**：AUC > 0.55、方向準確率 > 53%

## 第五階段：進階模型開發（第 9–10 週）
- XGBoost / LightGBM 優化
- 文本情緒模型整合
- 模型 ensemble
- **KPI**：AUC > 0.60、rank IC > 0.05

## 第六階段：風險引擎與成本引擎（第 11 週）
- 波動風險、尾部風險、流動性風險計算
- 交易成本模型建立
- Break-even move 計算邏輯
- **KPI**：風險指標覆蓋 5 類、成本估算誤差 < 15%

## 第七階段：回測框架建立（第 12 週）
- Walk-forward 回測引擎
- 多市場條件模擬（10 個場景）
- 績效指標自動化計算
- **KPI**：回測場景 ≥ 10 個、績效指標 ≥ 20 項

## 第八階段：可解釋決策層（第 13 週）
- B 軌決策轉譯邏輯
- SHAP 值可視化
- 標準化決策語言輸出
- **KPI**：決策分類一致性 > 85%

## 第九階段：前端介面開發（第 14–15 週）
- 市場趨勢頁、個股趨勢頁、決策頁
- 風險成本頁、報告生成功能
- 響應式設計
- **KPI**：頁面載入 < 3 秒、所有核心功能可操作

## 第十階段：後端部署與 API 服務（第 15–16 週）
- FastAPI / Flask 後端
- 資料庫設計（PostgreSQL / SQLite）
- 排程更新機制
- **KPI**：API 回應 < 500ms、可用性 > 99%

## 第十一階段：整合測試與品質保證（第 16–17 週）
- 端對端測試
- 模型漂移監控
- 效能壓力測試
- **KPI**：測試覆蓋率 > 80%、零重大 bug

## 第十二階段：上線發布與文件整理（第 17–18 週）
- 部署至 GitHub Pages / Vercel
- 使用者文件與免責聲明
- 期末報告與簡報製作
- **KPI**：系統穩定運行 72 小時、文件齊全
