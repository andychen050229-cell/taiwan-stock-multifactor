CRITICAL — 必須立即修復（5 項）

C1. val_pb 計算完全失效（99.99% 鎖死在 clip max）

發現者：資料科學家 + 量化分析師

檔案：engineer.py L390-414 ｜ 影響：val_pb 特徵喪失所有鑑別力

根因分析： val_pe_rank（L390）使用 groupby(ticker_col) 做時間序列排名而非 groupby(date_col) 做截面排名。 但 val_pb 的問題更根本：implied_shares = net_income / eps 在單位不匹配時（net_income 以千元計、EPS 以元計）， 算出的股數極小，導致 BVPS = equity / tiny_shares → 極大 → P/B = price / huge_bvps → 極小，最後被 clip(50) 截斷。
修復方案：
① 新增 implied_shares 單位校驗：若 median(shares) < 1000，可能單位不匹配，嘗試 ×1000 修正
② 若無法修正，改用 shares_outstanding 欄位（若資料源有提供）
③ 若以上都不可行，val_pb 移出 Feature Store 候選，避免後續誤用
④ 加入 diagnostic logging：clip 命中率 > 50% 時發出 ERROR

C2. val_pe_rank 錯誤：時間序列排名而非截面排名

發現者：量化分析師

檔案：engineer.py L390-392 ｜ 影響：val_pe_rank 是模型 top-1 重要特徵，定義錯誤直接影響預測品質

實測驗證：單一日期的 val_pe_rank 平均值 = 0.82（截面排名應為 ~0.5），確認為時間序列排名。
# 目前（錯誤）：每檔股票對自己的歷史 P/E 排名
df["val_pe_rank"] = df.groupby(ticker_col)["val_pe"].transform(lambda x: x.rank(pct=True))

# 修正：每個日期對所有股票的 P/E 做截面排名
df["val_pe_rank"] = df.groupby(date_col)["val_pe"].transform(lambda x: x.rank(pct=True, na_option="keep"))

C3. val_ev_ebitda 的 EV 計算：cash_proxy = current_assets 過於激進

發現者：分析師 + 資料科學家

檔案：engineer.py L461-466 ｜ 影響：82% 的 EV/EBITDA 在 (0,1] 區間，經濟學意義扭曲

根因：current_assets 包含存貨、應收帳款（佔總額 60-80%），不等於現金。EV 被大量低估。
修復方案：
① 優先使用 cash_and_equivalents 欄位
② 若無，改用 current_assets × 0.3（經驗比例）
③ 若以上都無，cash_proxy = 0（保守做法，不扣現金）

C4. Phase 2 Column Pruning 導致 Backtest 流動性篩選靜默失效

發現者：ML 工程師

檔案：執行Phase2_模型訓練.py L196-203 ｜ 影響：所有不流動股票被納入回測，回報可能虛高

根因：keep_cols 僅保留 selected_features + labels + meta columns。volume 和 turnover 未在 selected_features 中， 被 pruning 刪除。Backtest engine L88-103 找不到 volume → 靜默設定 _liquid=True，等於不做流動性過濾。
# 修正：擴充 keep_cols
keep_cols = (
    selected_features
    + [f"label_{h}" for h in horizons]
    + [f"fwd_ret_{h}" for h in horizons]
    + ["trade_date", "company_id", "close", "closing_price"]
    + ["volume", "turnover"]  # ← 新增：回測必需
)

C5. NOPAT 有效稅率：(OI-NI)/OI 包含非營業項目，且 fillna(0.2) 缺乏依據

發現者：分析師

檔案：balance_sheet_processor.py L243-252 ｜ 影響：NOPAT 可能偏差 10-30%

根因：(OI - NI) 包含利息、投資損益、匯兌損益等非營業項目。 0.2 fallback 對高科技免稅區企業偏高，對傳產企業偏低。
修復方案：
① 若有 income_tax_expense 欄位，直接用 eff_tax = income_tax / pre_tax_income
② 若無，改用台灣法定稅率 20% 但對 OI ≤ 0 設 0%，clip [0, 0.4]
③ NaN fallback 從 0.2 改為 0.17（台灣上市公司中位有效稅率）
④ 加入 per-industry 稅率表作為更精確的 fallback
