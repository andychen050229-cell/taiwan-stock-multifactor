# 資料來源與資料工程規範

## 資料來源分層

### A. 結構化資料
- **價量資料（OHLCV）**：yfinance / FinMind / TWSE OpenAPI
- **指數/類股報酬**：yfinance (^TWII, ^GSPC, ^IXIC)
- **市值資料**：yfinance / FinMind
- **財報資料**：FinMind（台股）/ yfinance（美股）
- **估值資料**：自行計算（P/E, P/B, EV/EBITDA）
- **分析師預估**：Alpha Vantage / FinancialModelingPrep（有限免費）
- **總經變數**：FRED API（利率、CPI、PMI、失業率）

### B. 文本資料
- **財經新聞**：NewsAPI / GDELT
- **法說會逐字稿**：SEC EDGAR（美股）/ 公開資訊觀測站（台股）
- **社群輿情**：PTT stock 板（輔助用途）

### C. 事件資料
- **財報公告日**：FinMind / yfinance earnings calendar
- **除權息**：FinMind / TWSE
- **重大事件**：手動標記 + 新聞分類模型

## 不可妥協的資料工程規則

### 規則 1：Point-in-time 對齊
- 財報資料以「公告日」而非「報告期」入庫
- 新聞以「發布時間」為準
- 絕不使用事後修正過的資料做歷史回測

### 規則 2：Corporate Actions 處理
- 所有價格做除權息調整（使用 adjusted close）
- 樣本宇宙包含已下市公司（避免生存偏誤）

### 規則 3：共線性治理
- 同來源欄位做 Pearson/Spearman correlation > 0.8 則保留一個
- 文本特徵做 embedding cosine similarity > 0.9 則合併
- 進行 VIF（Variance Inflation Factor）檢測

### 規則 4：標準化
- 市場內 cross-sectional z-score
- 產業內 rank normalization
- 時間序列 rolling z-score

### 規則 5：極端值與缺失值
- Winsorization at 1st/99th percentile
- 缺失值：forward fill（最多 5 日）後標記 flag
- 區分「未揭露」vs「不適用」vs「尚未公布」

### 規則 6：多 Horizon 視窗
| Horizon | Lookback 特徵 | Label |
|---------|--------------|-------|
| 短線 5D | 1–10 日 | 5 日後報酬方向 |
| 中線 20D | 20–60 日 | 20 日後相對報酬 |
| 長線 60D | 季/年頻 | 60 日趨勢評級 |

### 規則 7：來源可信度分級
| 等級 | 來源類型 | 可進入的層級 |
|------|----------|-------------|
| 高 | 交易所數據、官方財報 | 主證據層 |
| 中 | 專業新聞、分析師報告 | 輔助證據層 |
| 低 | 社群、論壇、自媒體 | 僅情緒/異常偵測 |

### 規則 8：版本控制
- 每次模型訓練記錄：Universe, Feature set, Cost assumptions, Data snapshot date
- 使用 Git + DVC 管理資料版本
