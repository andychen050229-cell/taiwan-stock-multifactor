## 0. Executive Brief（最上位指令）

**你（Claude Design）要做什麼？**
根據本文件的嚴格規格，為多因子股票分析系統製作一份**專業管顧風格的中文簡報**（目標 15–22 頁），**必須達到 BDA 2025 G10 等歷屆優良範例的視覺水準與論述密度**。

**三個不可妥協的原則：**

1. **資訊密度 + 結構清晰**：每一頁都有明確的 takeaway 放在右上角／頁首，副標題支撐論點，表格／圖表為證據。

```
  白底          = #FFFFFF  (主要背景)

```

```
  內文：思源黑體 Regular
```

- **頁首 Takeaway**：全頁最重要的一句話放這裡，28-32 pt 深藍粗體。例子：「D+20 預測訊號最穩定，xgboost 於所有指標勝出」。
- **左上：Action Title**（副標題），具體說明這頁要證明什麼
- **中央主視覺**：佔 60–75% 頁面空間
- **右側或下方：論點條列**：3–5 個 bullet，每個 10-25 字
- **頁尾**：資料來源、頁碼，灰色 9-10 pt

### 2.4 視覺元件

#### 表格設計

- 表頭：深藍底（`#1A5276`）+ 白字粗體
- 交替列：淺藍 `#F0F6FA` / 白底
- 邊框：淺藍 `#B0C4DE` 1px，**不用粗黑框**
- **最佳結果標紅**（G10 作法）：單一 cell 文字改為 `#C0392B` 粗體
- 數字靠右對齊、文字靠左對齊、標題置中
- **欄數上限**：6 欄（超過會變擁擠）
- **列數上限**：8 列（超過要分頁）

#### 圖表設計

- **Bar chart**：深藍主色 + 橘色強調最大條 / 最佳值
- **Line chart**：2–3 條線，深藍 + 橘色 + 綠色，線寬 2-3 px
- **流程圖**：方塊使用圓角 4 px，連接箭頭採用粗實線（不用虛線除非強調預期）
- **Isometric illustration**（G10 封面）：若有能力製作，可用於封面或章節分頁
- **禁用**：3D 圖表、餅圖超過 5 slice、雷達圖

#### 箭頭與流程

- **G10 黃色粗箭頭**（"1 → 2"）：可使用橘色 `#EA9D27` 作變形
- 水平流程用 `→`，垂直流程用 `↓`，每步驟保留等距留白

### 2.5 數字呈現規則

- **Hero number**：每頁最多一個「大數字 KPI」，用 60-80 pt 粗體，搭配 14-18 pt 說明
  - 範例：「**+10.53 pp**」大字 / 「Top 1% 選股 edge vs 基準」小字
- **百分比**：一律用 `%`，小數點後留 1-2 位（看重要度）
- **金額**：用千分位，單位（萬 / 億）明確標示
- **比較**：用 `↑` / `↓` 符號 + 顏色（綠 / 紅），不用文字「增加」「減少」
- **P 值**：學術場合保留 `p < 0.001` 標示，簡報場合可簡化為「✓ 顯著」

## 4. 不可違反的原則（Hard Rules）

### 4.1 內容原則

1. **每一頁必須有明確 Takeaway**。若你（Claude Design）無法為某頁寫出 1 句結論，該頁刪掉。
2. **數字必須來自真實資料檔案**。嚴禁使用佔位數字（placeholder）、美化後的假數字。所有 KPI 須能在 §1.2 原始資料檔案中找到來源。
3. **Source 必須標示**。每一頁下方灰字標註資料來源，例如「Source: `outputs/reports/threshold_sweep_xgb_D20.json`」
4. **中英文比例**：以中文為主（90%+），英文僅用於技術名詞（AUC、Sharpe、DSR、Walk-Forward CV 等）
5. **論述必須有時序**：Why → What → How → Results → Recommendations；章節之間不得跳躍。
6. **限制學術 jargon**：首次出現 jargon 時必須附上中文說明或簡短定義。

### 4.2 視覺原則

1. **留白比例 ≥ 30%**。每頁四周留邊、元件間 gap，**嚴禁排滿**。
2. **一頁一個 focal point**。不允許兩個視覺元件同時競爭讀者注意力。
3. **顏色功能化**：深藍 = 主結構 / 橘色 = 編號與強調 / 紅色 = 最佳結果 / 綠色 = 通過／正向 / 灰色 = 次要。
4. **禁止使用 emoji 裝飾**。資料圖表內的 ✓/✗ 符號可用（單色），但標題中不可出現 😊🚀📊 等。
5. **禁止 clipart 風格插圖**。插圖必須為 isometric 線條風格或 flat icon 風格。

### 4.3 格式原則

1. **檔案格式**：`.pptx` 或可匯出為 PDF 的設計格式；**4:3 比例優先 16:9**（教授投影機多為舊款）。
2. **字型嵌入**：若使用思源黑體，必須嵌入或轉換為大綱 / outline。
3. **可訪問性**：深藍背景上的白字對比度 > 7:1；橘色強調與白底對比度 > 4.5:1。
4. **單檔大小**：< 20 MB（避免教室投影機載入過慢）。

- **中間主視覺**：縱向 2-3 個條列重點 + 支撐的資料圖表
- **底部答案**：一句話結論用橘色粗體字強調
- **學習**：我們的頁 03 Why 可用「為什麼需要多因子系統？」疑問句

### 5.4 表格與紅色強調（G10 p.7「Chi-Square 特徵選擇比較」）
- **表格結構**：3 列 × 4 欄
- **最佳結果**：cell 文字改為紅色粗體（不是 cell 背景變色）
- **欄位名稱**：深藍底 + 白字
- **學習**：我們的頁 11 Model Performance、頁 12 DSR、頁 16 Top-K 都採用此手法

### 5.5 箭頭流程（G10 p.8 「1 → 2」粗黃箭頭）
- **箭頭**：寬度 60-80 px、飽滿黃色（或改為我們的橘色）
- **步驟標號**：箭頭上方 30 pt 深藍數字
- **學習**：我們的頁 05 Architecture Pipeline 與頁 20 Recommendation 都可用

## 6. 避免的常見錯誤（Anti-Patterns）

| 錯誤 | 為什麼錯 | 正確做法 |
|---|---|---|
| 每頁超過 50 字內文 | 讀者看不完 | 用 bullet + 短句（≤ 15 字） |
| 一頁塞 3 張圖表 | 焦點分散 | 一頁一個 focal point |
| 用全彩折線圖（> 5 條線） | 視覺混亂 | ≤ 3 條線，其他灰色弱化 |
| 用漸變背景 | 不專業 | 純色或極淺純色塊 |
| 表格無 alt row 著色 | 眼睛找不到行 | 交替 `#F0F6FA` / 白底 |
| Takeaway 寫在頁尾 | 讀者看不到 | 必置於頁首 |
| 直接用 Streamlit 截圖當配圖 | 解析度低 | 取數據重繪或 vector 替代 |
| 使用 default PowerPoint 模板 | 辨識度低 | 完全自訂版型 |
| 英文混中文無規則 | 閱讀障礙 | 技術詞英文 + 其餘中文 |

## 7. 製作流程建議（Workflow）

### Step 1: 沉浸式閱讀（1 小時）
- 逐頁閱讀 `多因子股票分析系統_完整研究報告_v11.5.17_2026-04-24.docx`（18 章）

### Step 2: 故事板繪製（30 分鐘）
- 拿白紙畫出 22 頁每頁的 Takeaway 與主視覺草圖
- 確認每頁有明確焦點 + 邏輯連結
- 刪掉「資訊重複」或「論點薄弱」的頁

### Step 3: 主題模板建立（1 小時）
- 在 Claude Design 建立：Cover 版型、Agenda 版型、內頁版型、結尾版型
- 定義 Color Tokens（見 §2.1）

### Step 4: 內容填入（3–4 小時）
- 依照 §3.1 的 22 頁順序填入
- 每頁完成後自我提問：「聽眾 10 秒內能記住 takeaway 嗎？」

### Step 5: 視覺精修（2 小時）
- 檢查每頁留白（≥ 30%）
- 統一字級與字距
- 圖表顏色套 color token

### Step 6: 審稿（1 小時）
- 對照 §8 Review Checklist 全項檢查
- 輸出 PDF 預覽（避免投影機字體偏移）
- 朗讀每頁 Takeaway，確認語意流暢
