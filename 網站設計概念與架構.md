## 1. 專案定位

**一句話**：以 2023/03 – 2025/03 台股歷史資料為基礎，整合 **9 支柱 1,623 候選因子**、
**LightGBM + XGBoost 雙引擎**、**Purged Walk-Forward CV** 與 **LOPO 深度驗證**
的「可回看、可解釋、可治理」的研究展示平台。

**明確不是**：即時交易系統、投資建議工具、大盤預測 SaaS。

**是**：學術課程研究專案的互動式快照終端機（terminal），定位貼近 Bloomberg Terminal
/ glint.trade / 財報狗，但以「歷史研究」而非「即時資料」為主軸。

## 2. 讀者雙軌與分頁策略

本站同時服務兩種讀者，**共用同一設計語言、但分頁對話**：

| 分軌 | 目標讀者 | 頁面 Icon | 文字調性 |
|---|---|---|---|
| 🌱 **投資觀察台** | 投資人、業界讀者、一般觀眾 | 🌱（暖色 amber 色調） | 白話中文、風險揭露前置、類比說明 |
| 🔬 **量化研究工作台** | 教授、評審、量化同行 | 📊📈🔬 等（冷色 blue/cyan 色調） | 密集 KPI、雙語代碼、monospace 數字、統計術語直白使用 |
| 🛡️ **治理 / 監控** | 風控、上線部署 | 🛡️📡🎯🔭 | 系統工程語調、health / snapshot / gate 等運維字彙 |
| 📖 **使用手冊** | 所有人 | 📖 | 教學敘事、流程圖、術語表 |

雙軌設計不是把一件事講兩遍，而是**從不同深度切入同一組事實**。投資人看
「2454 聯發科的命中率 66%」，研究員看「Hit Rate Edge +17.13pp @ D+20」。

## 3. 資訊架構與導覽

### 3.1 總體架構 — Option B Top-Nav

採用 `st.navigation(position="hidden")` + 自繪頂部 pill nav 的 **Option B** 架構：

```
┌──────────────────────────────────────────────────────────────────┐
│  [深色側邊欄]          [頂部 pill 導覽（sticky,每頁都有）]       │
│                        [該頁 Hero + 內容]                         │
│  - 品牌區塊                                                       │
│  - 系統健康度                                                     │
│  - 重整 / 手冊 按鈕                                               │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘
```

**為何採 Option B**：Streamlit Cloud 的 embed mode (`/~/+/`) 會隱藏側邊欄，
預設側邊欄導覽在 Cloud 就失效。把主導覽搬到 main canvas 頂部，**不管側邊欄
是否顯示都有完整導覽**。

### 3.2 Top-Nav 分組（共 12 pills，手冊另由側邊欄按鈕進入）

```python
TOP_NAV_GROUPS = {
    "總覽":      [home, interpret],                   # 情境主控台 / 投資觀察台
    "個股研究":  [data, icir, text],                   # 企業體檢 / 信號穩定性 / 影響鏈情緒
    "量化引擎":  [model, feature, backtest, phase6],   # 模型績效 / 特徵工程 / 策略回測 / Phase 6 深度
    "治理監控":  [governance, signal, extended],      # 模型治理 / 信號監控 / 擴充分析
}
```

### 3.3 側邊欄元素（由上而下）

1. **品牌區塊** — `股票預測系統` logo + 版本
2. **系統健康度卡** — 9/9 gates ✓（動態讀取 phase2_report）、資料期間、樣本數、特徵數、DSR 值、最後驗證時間
3. **真實動作按鈕** — 🔄 重新整理（`st.cache_data.clear()` + rerun）、📖 使用手冊（`st.switch_page`）
4. **配色說明** — 賽博龐克深色底 `linear-gradient(180deg, #0a1420 → #121c2b → #0f1a28)`，
   邊框 cyan 0.08 alpha，隱藏 Streamlit 自動 nav（`position="hidden"` + `display:none`）

## 4. 視覺設計概念 — "glint-light"

### 4.1 核心理念：雙情緒融合

> **側邊欄深（技術自信）＋ 主畫布亮（可讀清晰）＝ 一套視覺，兩種情緒。**

- **側邊欄**：Bloomberg Terminal 質感。深色底、cyan/indigo 點綴、mono 字體。
  告訴讀者「這是一個嚴謹的工程系統」。
- **主畫布**：off-white `#fafbfc` + 40×40 科技網格 + 左上藍右上紫的徑向光暈。
  告訴讀者「這裡的內容可親近、可閱讀」。

兩種情緒用同一組 token 連結（`--gl-blue`、`--gl-grad-tech`、`--gl-shadow-glow`），
所以不會打架。

### 4.2 設計參考

- **glint.trade** — gradient hero、KPI 卡、live 指示器
- **Bloomberg Terminal** — 深色側邊欄、mono 字、密集 chip
- **財報狗** — 白話中文、圖表留白

### 4.3 為什麼不全深色 / 不全亮

- 全深色：長時間讀表格、中文字、密集數字容易疲勞
- 全亮：缺少「這是專業工程工具」的暗示
- 混搭：**主內容亮**符合閱讀、**側邊欄深**符合品牌，兩者交界是 1px cyan 0.08 border

## 6. 六大科技感設計技法

### 技法 1：Glass-morphism 玻璃感面板

```css
background: rgba(255,255,255,0.72);
backdrop-filter: blur(8px);
border: 1px solid rgba(226,232,240,1);
```

**應用**：sticky topbar、hero `.gl-panel-tint` 變體

### 技法 2：Tech Grid 科技網格（signature 簽名手法）

```css
body::before {
  content: "";
  position: fixed; inset: 0; z-index: 0;
  background-image:
    linear-gradient(rgba(37,99,235,0.025) 1px, transparent 1px),
    linear-gradient(90deg, rgba(37,99,235,0.025) 1px, transparent 1px);
  background-size: 40px 40px;
  pointer-events: none;
}
```

**應用**：每一頁都有；40px 網格、0.025 alpha 極淡、固定不隨捲動

### 技法 3：Gradient 標題（Hero clip-text）

```css
.gl-hero-title {
  background: var(--gl-grad-hero-title);
  -webkit-background-clip: text;
  -webkit-text-fill-color: transparent;
}
```

**應用**：每頁 hero 標題；**只用這個漸層**，不要在其他地方用 clip-text

### 技法 4：Pulse LIVE 指示器

```css
@keyframes gl-pulse {
  0%   { box-shadow: 0 0 0 0    rgba(16,185,129,.6); }
  70%  { box-shadow: 0 0 0 8px  rgba(16,185,129,0); }
  100% { box-shadow: 0 0 0 0    rgba(16,185,129,0); }
}
.gl-live-dot { animation: gl-pulse 2s cubic-bezier(.4,0,.6,1) infinite; }
```

**應用**：系統健康度的綠點、topbar 的 LIVE chip

### 技法 5：Mono Tabular Numerics（數字對齊）

```css
.gl-mono, .gl-kpi-value, td, th {
  font-family: 'JetBrains Mono', 'Noto Sans TC', monospace;
  font-variant-numeric: tabular-nums;
  font-feature-settings: "tnum" 1, "zero" 1;
}
```

**應用**：**所有**數字、ticker、bps、AUC、IC、%、金額、樣本數

### 技法 6：左側 Accent Bar（h2 / insight box）

```css
.block-container h2::before {
  content: ""; position: absolute;
  left: 0; top: 4px; bottom: 4px;
  width: 4px;
  background: var(--gl-grad-pri);
  border-radius: 4px;
}
```

**應用**：每個 `st.header()` 都自動有；insight-box 也用 4px 左邊框（顏色對應 intent）

## 7. 核心元件規格

共 10 個元件，都在 `utils.py` 中。

### 7.1 GlHero — `render_hero()`
- 結構：eyebrow chip + gradient title + subtitle + meta chips
- 背景：`linear-gradient(135deg, #fff 0%, #f8fafc 100%)` + 右上 radial 光暈
- 左上可選 `::before` 光球（`filter: blur(40px)`）

### 7.2 GlKpi — `render_kpi(label, value, delta, sub, accent)`
- 3px 頂邊框（顏色變體：blue / violet / cyan / emerald / amber / rose）
- label: 0.72rem UPPERCASE letter-spacing 0.08em
- value: 2rem JetBrains Mono 700 tabular-nums
- delta: mono 0.7rem，`up/down/neu` 三色
- hover: border 變 cyan、shadow-glow、translateY(-1px)

### 7.3 GlPanel — `.gl-panel` / `.gl-panel-tint`
- 白底、14px radius、1px border、shadow-sm
- tint 變體：`linear-gradient(135deg, #f8fafc 0%, #ffffff 100%)`

### 7.4 GlChip — `.gl-chip` + `.primary / .violet / .ok / .warn / .danger`
- 999px pill、11px 字、mono
- 有 border + bg tint（alpha 0.08 / 0.2）

### 7.5 GlBox — `.insight-box / .warning-box / .success-box / .gl-box-danger`
- 左 4px accent bar + 漸層 tint 底
- radius 非對稱 `0 10px 10px 0`（讀起來像旗子）
- 四個 intent：info(blue) / warn(amber) / ok(emerald) / danger(rose)

### 7.6 GlTopbar — `render_topbar(crumb, chips, show_clock)`
- sticky 頂部、麵包屑 + model chips + JetBrains Mono 即時時鐘
- 玻璃感背景 + blur(8px)

### 7.7 GlTopNav — `render_top_nav(groups, active_page_title)`
- Option B 核心元件
- 繪製分組 label（eyebrow 風格）+ pill nav 列
- 當前頁以 `aria-current="page"` 驅動深色高亮

### 7.8 GlSystemHealth — `inject_sidebar_health(gates_passed, total_gates, ...)`
- 品質閘門圓形 cyan ring gauge + 89% 等 mono 數字
- `gates_passed / total_gates` 動態（讀 `load_quality_gates()`）

### 7.9 GlPhaseTimeline — `render_phase_timeline(current_phase)`
- 6 相 horizontal timeline
- 當前相用 `--gl-grad-pri`，其他綠色
- 左 3px accent bar

### 7.10 GlAucGauge — `render_auc_gauge(val, min, max, label)`
- SVG 半圓儀表、blue→violet 漸層、tick 在 min/mid/max

## 8. 圖表與表格規範

### 8.1 Plotly 統一模板

所有 Plotly 圖表**必須**套用：

```python
from utils import glint_plotly_layout, glint_heatmap_colorscale, glint_colorbar

fig.update_layout(**glint_plotly_layout(
    title="標題",
    subtitle="說明這張圖在看什麼",   # 副標必加
    height=360,
    xlabel="...", ylabel="...",
))
```

- **paper_bgcolor / plot_bgcolor**: `rgba(0,0,0,0)` 透明（讓 tech grid 穿透）
- **font**: Inter sans（標題）+ JetBrains Mono（tick）
- **colorway**: 8 色 token palette（blue/violet/cyan/emerald/amber/rose/indigo/pink）
- **hoverlabel**: 深色 `#0f1419` + cyan 邊框（反差給閱讀性）
- **grid**: `rgba(37,99,235,0.06)` 極淡藍
- **tick**: 10px mono、`#64748b`

### 8.2 色階（Heatmap）

- `glint_heatmap_colorscale("diverging")` — **預設**。rose → neutral → emerald（綠=好）
- `glint_heatmap_colorscale("cool")` — 藍系 sequential（數量 / 強度）
- `glint_heatmap_colorscale("warm")` — 琥珀系 sequential（風險 / 警告）
- **禁用** Plotly 預設 `RdYlGn` / `Blues` / `Viridis`，只用 token 色階

### 8.3 表格

- `st.dataframe` / `st.table` 自動套 `.gl-table` 風格：
  - Header: `tint` 底 + mono UPPERCASE 0.72rem letter-spacing 0.08em
  - Body: mono tabular-nums 0.84rem
  - Hover: row 底色 `rgba(37,99,235,0.03)`
- 數字欄位語義著色：
  - `.cell-up` emerald · `.cell-down` rose · `.cell-ticker` blue · `.cell-dim` slate-3

## 10. 文字語調與雙語規範

### 10.1 語言主次
- **繁體中文 zh-TW 為主**，英文技術術語可自由 code-switch（`AUC`、`ICIR`、`Purged WF`、`Feature Store`、`embargo`）
- **雙語 pipe 標題**：`模型指標分析 | Model Metrics`、`選擇 Horizon | Select Horizon`
- **台灣金融術語保留**：出手率、籌碼、支柱、回測、命中率

### 10.2 人稱
- 用「本平台 / 本系統 / 本專案」第三人稱自稱
- 不用「您 / 你」對話口吻
- 不用「我們 / 我」主觀口吻

### 10.3 tone 三大原則
1. **Confident but hedged** — 給數字但給信賴區間；說結論但給 DSR / Bootstrap 結果
2. **Researcher-to-peer** — 預設讀者有統計素養；投資人面才向下翻譯
3. **No marketing adjectives** — 用「嚴謹 · 可回看 · 可解釋 · 可治理」，不用「頂尖 · 世界級 · 革命性」

### 10.4 標點
- `,` 中文句內不用，句末用 `。`
- 英文縮寫後用全形括號：`LOPO（Leave-One-Pillar-Out）`
- 箭頭用 `→`（U+2192）、破折號用 `—`（em dash）、分隔用 `·`（middle dot）

### 10.5 數字格式
- 百分比：2 位小數帶正負號 `+13.83% / −8.70%`（ASCII `%`, 不用 `％`）
- bps：`+1.5bps`
- 比率：3–4 位小數 `AUC 0.6455 · IC 0.015`
- 千位分隔：`948,976 rows · 1,930 家`
- **全部強制** `font-family: JetBrains Mono` + `tabular-nums`

### 10.6 Emoji 用法
- **結構性、非裝飾性**
- 每個 tab 只有 1 個結構 emoji（🌱 📊 📈 💰 🔬 🗃️ 🛡️ 📡 🎯 📝 🔭）
- 狀態用 `✅ ❌ ⚠️ 📋 📅 🎯 ⚡ 🔒`
- **不在 body copy 用** emoji、**不連用兩個**、**不當項目符號**

### 10.7 範例（逐字可搬用）

> 以 **2023/03 – 2025/03** 台股歷史資料為基礎，整合 **9 支柱 1,623 候選因子**、
> **LightGBM + XGBoost 雙引擎**、**Purged Walk-Forward CV** 與 **LOPO 深度驗證**，
> 提供可回看、可解釋、可治理的研究展示平台。

> ⚠️ **免責聲明**：本系統為課程專案成果，不構成任何投資建議。所有顯示結果均為歷史回測數據。

> 近期回撤僅 1%，價格走勢相對穩定；目前處於高波動環境，短期價格變動可能較劇烈。

## 11. 品質門檻與單一真相來源

### 11.1 問題背景
先前 8 個位置硬編碼 `9/9` / `Phase 2 全數通過`，當實際 gate 變動時會產生沉默矛盾。

### 11.2 解方：`load_quality_gates()`
位於 `utils.py`，回傳：
```python
{
    "passed": int,          # 通過數
    "total": int,           # 總數
    "all_pass": bool,       # passed == total
    "last_verified": str,   # ISO datetime
    "failed_names": list,   # 失敗 gate 名稱
}
```

來源：最新的 `outputs/reports/phase2_report_*.json`。

### 11.3 單一真相套用點（全部都已遷移）
1. `app.py` 側邊欄 health 卡 `gates_passed / total_gates`

3. `6_Model_Governance.py` 標題 caption + 狀態判斷

### 11.4 Fallback 策略
報告不存在時回退到 `9/9`（避免顯示 `0/0`），但 `last_verified` 用 `None`
以提示讀者「這是 fallback」。

## 12. 非功能要求

### 12.1 Streamlit Cloud 相容
- `st.Page(page_file=absolute_path)` — **絕對路徑**（Cloud 的 shim entrypoint 會讓相對路徑壞掉）
- 避免 `position="sidebar"` 作為唯一導覽（embed mode 會隱藏）
- 所有 `st.cache_data` 加 `ttl=3600`（每小時刷新）

### 12.2 字體 Fallback
```
Inter → Noto Sans TC → Microsoft JhengHei → PingFang TC → Segoe UI → sans-serif
```
Cloud 不保證 Inter 可載入，CJK fallback 保證中文不會變 tofu。

### 12.3 色彩對比
- 主畫布黑底：AAA（WCAG 7:1）
- 側邊欄白字黑底：AAA
- accent text on tint：AA（4.5:1）必須

### 12.4 響應式斷點
- `≥ 1200px`：full 6-up KPI grid、3-up phase timeline
- `900–1199px`：3-up KPI、phase 3+3 二列
- `< 900px`：side menu hidden、research card 全寬

### 12.5 Plotly 效能
- 每頁 `@st.cache_data(ttl=3600)` 套件級快取
- 大型 figure（> 1000 points）用 `fig.update_traces(mode="lines")`（移除 markers）
- 絕不 `config={"displayModeBar": True}` 汙染工具列
