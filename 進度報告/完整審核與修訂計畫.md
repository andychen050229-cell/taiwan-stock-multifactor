## 0. 審查前提與判讀原則

1. 本專案的正確定位是：**固定歷史資料集下的股市趨勢預測導向決策輔助系統**，不是即時投顧產品，也不是自動交易系統。
2. 雙面板設計（投資解讀面板 / 量化研究工作台）應被視為**核心產品設計**，不是附加頁面。  
   - 投資解讀面板：把模型結果翻譯成一般投資者可理解的決策語言。  
   - 量化研究工作台：保留完整方法論、指標、圖表與檢驗細節。
3. 目前 Phase 3 已公開落地：repo 內已有 `run_phase3.py`、`dashboard/pages/6_️_Model_Governance.py`、`7__Signal_Monitor.py`、`outputs/governance/*.json`、`outputs/reports/phase3_report_*.json`。  
4. 但「已完成 Phase 3」不代表「所有頁面都已收斂為單一一致敘事」。本報告重點不是否認完成，而是找出**尚未完全對齊、尚未完全 polish、或尚未完全研究誠實化**的地方。
5. 若某問題明顯屬於「Phase 3 規劃已存在、但頁面尚未完全串接」，本報告會標示為：
   - **整合不足（Integration Gap）**
6. 若某問題已直接影響研究可信度、使用者理解或產品誠信，本報告會標示為：
   - **高優先問題（P0 / P1）**

# 第一部分：總體結論（Executive Verdict）

## 1.1 總評

### 結論一句話
這個網站已經不是「未完成的學生作業」，而是一個**骨架很強、研究方法明確、Phase 3 已初步產品化落地**的展示系統；但若以**嚴格的資料科學家、UI/UX 設計師、股票分析師、投資使用者**四種視角交叉檢驗，**目前仍不能說所有頁面都已經足夠完善**。

### 原因摘要
目前最大的問題不是「功能不夠多」，而是以下四件事尚未完全收斂：

1. **投資解讀面板的語義與資料來源仍有方法論風險**  
   尤其是「模型信心度／預測報酬排序」與 `fwd_ret` / `label` / 歷史結果之間仍存在容易混淆的呈現方式。

2. **Phase 3 雖已落地，但整站仍存在 Phase 2 / Phase 3 資料來源與敘事層的碎片化**  
   有些頁面仍主要吃 phase2 report；有些頁面已直接吃 governance JSON；首頁與 README 也未完全同步。

3. **固定歷史資料集的定位雖已寫清楚，但個別頁面仍殘留「像在持續更新」或「像即時推薦」的訊號**  
   這會讓嚴格評審質疑系統究竟是 historical snapshot 還是 live product。

4. **部分頁面已有高完成度，但仍缺乏最後一層「研究誠實化＋產品一致化」收尾**  
   也就是：同一套 truth，是否在首頁、投資解讀面板、專業頁、治理頁、README 中都說同一件事。

## 2.2 投資解讀面板（原智慧選股看板）

### 目前優點
1. 是整站最具產品感、最能被一般人理解的一頁。
2. 已涵蓋使用者在意的資訊：
   - 前五名
   - 公司背景
   - 基本面摘要
   - 技術圖
   - 交易成本
   - 多週期比較
   - 風險提醒
3. 已補上頂層 exception handling，較早期審核報告指出的崩潰風險已有改善。

### 關鍵問題（最高優先）
#### 問題 1：prediction / historical outcome 尚未乾淨切割
目前頁面文案與邏輯仍混用下列概念：
- 模型信心度
- 預測報酬排序
- `label_{horizon}`
- `fwd_ret_{horizon}`
- 歷史觀察報酬

這會導致：
- 一般使用者把事後值誤解成當下預測值
- 嚴格評審質疑頁面是否存在 ex post 呈現風險

#### 問題 2：信心等級目前不是來自真正的 model probability / calibrated score
若信心等級仍由 `abs(fwd_ret)` 或類似歷史結果衍生，就不能稱為模型信心度。

#### 問題 3：頁面目前更像「推薦頁」，而不是「判讀頁」
這與專案真正定位（決策輔助，不是直接投資建議）仍有張力。

### 必修改善
#### A. 直接把頁面改名與改語義
- 名稱：**投資解讀面板**
- 不再主打「推薦」
- 主打：**歷史快照下的模型判讀示範**

#### B. 拆成兩種模式（必要）
1. **Historical Review Mode**
   - 可顯示 `fwd_ret_horizon`
   - 但必須標記為：
     - 歷史觀察值
     - 僅供回看模型表現
     - 非當時可知資訊

2. **Prediction Snapshot Mode**
   - 只能顯示：
     - calibrated probability
     - top class
     - confidence bucket
     - model score
   - 不顯示事後 forward return

#### C. 所有排序邏輯改為 model score / calibrated probability
- 不得再使用 `fwd_ret_horizon` 當作當前主要排序依據
- 若目前產物缺少 explicit score，則：
  - 先在頁面明確寫：本頁為 historical snapshot demo
  - 後續從 Phase 3 prediction pipeline 產出 snapshot prediction artifact

#### D. 信心系統重寫
信心來源應為：
- top-1 probability
- top-1 與 top-2 probability margin
- calibration 後 bucket

可設計：
- 高信心：P(top class) >= 0.70 且 margin >= 0.20
- 中信心：P(top class) 0.58–0.70
- 低信心：其餘
- 中性／觀望：margin 過低或 signal 被靜默條件抑制

#### E. 新增「不誤導」機制
每張股票卡片加上：
- 適用時間尺度：D+1 / D+5 / D+20
- 風險語氣：偏高 / 中 / 低
- 為何被納入：Top Drivers（可接 SHAP / top features）
- 何時不應該相信：若 drift / low confidence / liquidity issue / high cost erosion
